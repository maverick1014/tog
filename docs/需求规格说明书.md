## 1. 项目概述

本系统是一套**面向教会的网页管理系统**，用于管理成员、聚会出席、奉献、培训，
以及特色的**「四十天一对一守望」门徒培育流程**。

- **使用对象**：牧师、小组长、副组长、行政同工等教会管理者。
- **界面语言**：**全中文界面**（面向中文用户）。
- **交付形态**：Web 网页应用 + 后端 API + 云数据库。

### 1.1 目标

1. 用一个系统统一管理教会「人、聚会、奉献、培训、门训」五大板块。
2. 成员按**教会身份角色**分层管理（牧师 / 小组长 / 副组长 / 实习组长 / 核心成员 / 普通成员 / 新成员）。
3. 每位成员都有**个人培训档案**，记录参加过哪些培训及进度。
4. 培训**高度可自定义**（可多场次、可指定讲师、可决定谁能报名）。
5. 支持「四十天一对一守望」的**级联式门训**，牧者可**实时监控**每个人的进度。

## 4. 功能模块

### 4.1 成员管理（成员目录）

**目标**：集中管理教会所有成员信息。

**成员字段**：
- 姓名（英文名 / 中文名）
- 联系方式（邮箱、电话）
- 性别、出生日期
- **身份角色**（见第 3 节）
- 状态（在册 / 停止聚会）
- 所属小组、所属家庭
- 加入日期、备注

**功能**：
- 成员列表：支持按**角色**、**小组**筛选，按姓名搜索。
- 新增 / 编辑 / 删除成员。
- **成员详情页**：展示基本信息 +**个人培训档案**（见 4.6）+ 参与的门训对子。

### 4.2 小组管理

- 小组列表（名称、简介、组长）。
- 每个小组可查看**组员名单**。
- 指定小组长（关联到成员）。

### 4.3 家庭管理（可选）

- 以「家庭」为单位归类成员（同一地址 / 同一家庭）。
- 家庭详情展示家庭成员列表。

### 4.4 聚会与出席

**目标**：安排聚会并记录出席。

**聚会字段**：标题、类型（主日崇拜 / 聚会 / 祷告会 / 团契 / 其他）、地点、开始时间、结束时间、说明。

**功能**：
- 聚会列表（按时间排序，区分即将到来 / 已过去）。
- 新增 / 编辑 / 删除聚会。
- **出席点名**：在聚会详情页对每位成员标记**出席 / 缺席 / 请假**，可一次性批量保存。
- 出席统计（某场聚会出席人数）。

### 4.5 奉献管理

**目标**：记录奉献并统计。

**奉献字段**：奉献人（可匿名）、金额、币种、类别（十一奉献 / 主日奉献 / 建堂 / 宣教…）、
方式（现金 / 银行转账 / 刷卡 / 线上 / 其他）、奉献日期、备注。

**功能**：
- 奉献记录列表，可按成员、类别筛选。
- 新增 / 编辑 / 删除奉献。
- **奉献汇总**：按类别统计总额、总奉献额。

### 4.6 培训管理（重点）

培训分三部分：**培训目录**、**报名与出席**、**个人培训档案**。

#### 4.6.1 培训目录（可高度自定义）

**培训字段**：
- 名称、说明、类别
- **讲师 / 带领者**（关联到成员）
- **场次数量**（一次或多次；例如三周三堂课）
- **是否开放报名**（`is_enrollable`）
- 开始日期、结束日期

**多场次（Session）**：一个培训可有多个场次，每个场次有场次编号、标题、时间、地点、备注
——**满足「同一培训可能有多次」的需求**。

#### 4.6.2 报名与审核

- 成员可**报名**培训（若该培训开放报名）；初始状态为**待审核（pending）**。
- **管理员 / 讲师审核**：决定是否通过（approved），并可标记进行中、完成、退出。
- 状态：待审核 / 已通过 / 进行中 / 已完成 / 已退出。

#### 4.6.3 出席点名与「生成名单核对」

- **管理员决定每位报名者在每个场次是否出席**（attended 是 / 否）。
- 系统可**生成核对名单（namelist）**：以「报名成员 × 各场次」的表格呈现，方便逐一点名核对。
- 用于满足「if enrolled → 由管理员决定是否出席，并可生成名单核对」的需求。

#### 4.6.4 个人培训档案

- 每位成员在**成员详情页**都能看到**自己参加过的所有培训及进度**（完成百分比、状态）。
- 满足「每个成员应有其培训档案，记录参加过的培训及进度」的需求。

### 4.7 四十天一对一守望（门徒培育 · 特色模块）

> 需求原文精神：牧者先教小组长，小组长再教副组长，被教过的人可以再教下一个人，
> 直到所有不同成员都上过这个课程；课程共 40 天，**带领者自己填写表单**更新被培训者状态，
> **牧者可通过系统实时查看每位成员的进度**、随时监督掌控。

#### 4.7.1 核心概念

- **门训计划（Program）**：如「四十天一对一守望 2026」，含总天数（默认 **40 天**）。
- **一对一对子（Pair）**：`带领者(mentor) → 被带领者(trainee)`。
- **级联关系（cascade）**：某个对子的「被带领者」，会成为下一个对子的「带领者」，
  形成一条培育链（牧者 → 小组长 → 副组长 → …）。系统用 `parent_pair_id` 记录这条链的上下级来源。

```
牧者 ──带领──▶ 小组长 ──带领──▶ 副组长 ──带领──▶ 核心成员 ──▶ …
（对子1）        （对子2）         （对子3）
```

#### 4.7.2 每日表单（带领者填写）

- 每个对子有 **40 天**的进度格子（Day 1 ~ Day 40）。
- **带领者**在系统里逐天填写：某天是否完成、备注 / 反馈。
- 一个「一对一对子 + 天数」唯一，重复填写为更新（幂等）。

#### 4.7.3 牧者总览（实时监控）

- 牧者可看到**整个计划下所有对子**的完成进度（已完成天数 / 40、完成百分比）。
- 一屏掌握全局：谁进度快、谁落后，随时监督掌控。
- 由数据库视图 `discipleship_pair_summary` 实时汇总，无需手动统计。

## 5. 数据库设计（Schema 概览）

以下为**以数据库为事实来源**的核心数据表（详见 `supabase/migrations`）。

### 5.1 人与组织
- `members`：成员（含身份角色枚举 `member_role`、状态、所属小组 / 家庭）
- `groups`：小组（含组长外键）
- `households`：家庭

### 5.2 聚会与奉献
- `events`：聚会（类型、时间、地点）
- `event_attendance`：聚会出席（出席 / 缺席 / 请假），`(event_id, member_id)` 唯一
- `donations`：奉献（金额、类别、方式、日期）

### 5.3 培训
- `trainings`：培训目录（讲师、场次数、是否开放报名）
- `training_sessions`：培训场次（支持一个培训多个场次）
- `training_enrollments`：报名记录（状态、进度 0–100），`(training_id, member_id)` 唯一
- `training_attendance`：每场次出席（用于生成核对名单），`(session_id, member_id)` 唯一

### 5.4 四十天守望
- `discipleship_programs`：门训计划（总天数，默认 40）
- `discipleship_pairs`：一对一对子（带领者、被带领者、`parent_pair_id` 级联、状态）
- `discipleship_progress`：每日进度表单（`(pair_id, day_number)` 唯一，含完成状态与备注）
- `discipleship_pair_summary`（视图）：对子完成度实时汇总，供牧者总览

### 5.5 枚举类型
`member_role`、`member_status`、`gender_type`、`event_type`、`attendance_status`、
`donation_method`、`enrollment_status`、`pair_status`。

## 6. API 设计（REST · 前缀 `/api`）

| 模块 | 主要接口 |
| --- | --- |
| 成员 | `GET/POST /members`、`GET/PATCH/DELETE /members/:id`、`GET /members/:id/trainings`（个人培训档案）|
| 小组 | `GET/POST /groups`、`GET/PATCH/DELETE /groups/:id` |
| 家庭 | `GET/POST /households`、`GET/PATCH/DELETE /households/:id` |
| 聚会 | `GET/POST /events`、`GET/PATCH/DELETE /events/:id`、`POST /events/:id/attendance`（批量点名）|
| 奉献 | `GET/POST /donations`、`GET /donations/summary`、`PATCH/DELETE /donations/:id` |
| 培训 | `GET/POST /trainings`、`GET/PATCH/DELETE /trainings/:id`、`GET /trainings/:id/namelist`（核对名单）|
| 培训场次 | `POST /trainings/:id/sessions`、`PATCH/DELETE /trainings/sessions/:sessionId`、`POST /trainings/sessions/:sessionId/attendance` |
| 报名 | `POST /trainings/:id/enroll`、`PATCH/DELETE /trainings/enrollments/:enrollmentId`（审核 / 进度）|
| 门训 | `GET/POST /discipleship/programs`、`GET /discipleship/programs/:id/overview`（牧者总览）|
| 门训对子 | `GET/POST /discipleship/pairs`、`GET/PATCH/DELETE /discipleship/pairs/:id`、`POST /discipleship/pairs/:id/progress`（每日表单）|

## 8. 非功能性需求

- **界面语言**：全中文（简体），可预留多语言扩展。
- **响应式**：适配桌面为主，兼容平板 / 手机基本浏览。
- **数据一致性**：以数据库 Schema 与迁移文件为准；唯一约束防止重复报名 / 重复点名。
- **可维护性**：Monorepo + 共享类型；前后端类型一致。
- **可扩展**：预留登录认证（Supabase Auth）与按角色权限控制。
